# 项目架构文档

## 概述

本项目是一个使用Rust编写的x86汇编测试框架，用于为rv环境提供基准测试数据。项目直接在原生x86环境下执行汇编代码来验证其正确性，并生成包含执行结果的输出文件。

## 技术架构

### 核心组件

1. **汇编解析器**
   - 解析包含嵌入式JSON配置的汇编文件
   - 提取CONFIG块中的JSON配置信息
   - 处理寄存器初始值、内存区域和内存数据配置

2. **编译系统**
   - 使用NASM汇编器将汇编代码编译为目标文件
   - 根据Mode配置确定编译模式（32位或64位）
   - 使用系统链接器生成可执行的ELF文件

3. **原生执行环境**
   - 使用mmap分配执行内存
   - 直接在x86原生环境下执行汇编代码
   - 精确执行到HLT指令停止

4. **寄存器捕获器**
   - 捕获所有寄存器的最终状态
   - 生成包含执行结果的新汇编文件

### 技术栈

- **编程语言**: Rust
- **汇编器**: NASM (Netwide Assembler)
- **链接器**: x86_64-linux-gnu-ld (或相应的32位链接器)
- **JSON解析**: serde_json (Rust crate)
- **内存管理**: libc crate (用于mmap系统调用)
- **执行环境**: 原生x86硬件/操作系统
- **操作系统**: Linux (支持x86/x64架构)

## 数据流

### 1. 输入处理阶段
1. 读取汇编源文件 (.asm)
2. 解析CONFIG块中的JSON配置
3. 提取寄存器初始值、内存区域和内存数据配置

### 2. 编译阶段
1. 根据Mode配置确定编译模式
2. 使用NASM编译汇编代码为目标文件
3. 使用链接器生成可执行的ELF文件

### 3. 执行阶段
1. 解析ELF文件获取指令和数据
2. 创建父子进程，父进程使用ptrace控制子进程执行
3. 在子进程中使用mmap分配固定地址的代码和数据内存
4. 将指令加载到分配的代码内存中
5. 根据配置初始化数据内存
6. 在起始和结束位置设置int3断点
7. 在第一个断点处清空寄存器并设置初始状态
8. 继续执行直到遇到结束断点
9. 通过ptrace读取最终寄存器状态

### 4. 结果处理阶段
1. 捕获所有寄存器的最终状态
2. 生成包含执行结果的新汇编文件

## JSON配置格式

### 核心字段

- **RegData**: 寄存器数据配置
  - 通用寄存器 (RAX, RBX, RCX, RDX, RSI, RDI, RSP, RBP, R8-R15)
  - XMM寄存器 (XMM0-XMM15)
  - 标志寄存器 (FLAGS)

- **Mode**: 执行模式
  - "32BIT": 32位执行模式
  - 省略: 64位执行模式（默认）

- **MemoryRegions**: 内存区域配置
  - 键: 内存起始地址
  - 值: 内存区域大小

- **MemoryData**: 内存数据配置
  - 键: 内存地址
  - 值: 要写入的数据数组

## 内存管理

### 内存分配
- 使用mmap系统调用分配执行内存
- 根据MemoryRegions配置分配指定内存区域
- 为代码段分配可执行内存 (PROT_READ | PROT_EXEC)
- 为数据段分配可读写内存 (PROT_READ | PROT_WRITE)

### 数据初始化
- 根据MemoryData配置初始化内存内容
- 支持十六进制字符串和数字格式的数据

### 内存释放
- 执行完成后使用munmap释放所有分配的内存

## 寄存器支持

### 通用寄存器
- 64位: RAX, RBX, RCX, RDX, RSI, RDI, RSP, RBP, R8-R15
- 32位: EAX, EBX, ECX, EDX, ESI, EDI, ESP, EBP, R8D-R15D

### SIMD寄存器
- XMM寄存器: XMM0-XMM15 (128位)
- YMM寄存器: YMM0-YMM15 (256位)

### 标志寄存器
- FLAGS/EFLAGS/RFLAGS寄存器支持

## 执行环境

### 原生x86执行
- 直接在x86硬件上执行，不使用任何模拟器
- 使用父子进程协作模型，父进程通过ptrace系统调用控制子进程执行
- 使用mmap分配固定地址的可执行内存和数据内存
- 通过int3断点精确控制执行流程
- 精确控制执行到HLT指令停止

### 执行控制
- 使用ptrace系统调用实现精确执行控制
- 在起始和结束位置设置int3断点
- 寄存器状态完整捕获
- 内存状态可控初始化
- 支持XMM/YMM寄存器状态清零和捕获

## 输出格式

生成的汇编文件保持与输入文件相同的格式，但包含执行后的寄存器状态：

```asm
%ifdef CONFIG
{
    "RegData": {
        "RAX": "0x000000000000000c",
        "RBX": "0x0000000000000000",
        // ... 其他寄存器状态
    }
}
%endif

// 原始汇编代码保持不变
mov rdx, 0xc0
shr rdx, 4

hlt
```

## 依赖关系

### 系统工具
- NASM: 汇编器
- x86_64-linux-gnu-ld: 链接器
- 原生x86执行环境

### Rust Crates
- serde_json: JSON解析
- libc: 系统调用接口 (mmap, munmap等)
- goblin: ELF文件解析 (可选)

## 扩展性设计

### 模块化架构
- 模块化设计便于扩展新功能
- 可插拔的执行环境支持
- 灵活的配置解析机制

- 通过JSON配置支持复杂测试场景
- 支持不同的执行模式(32位/64位)
- 可自定义内存布局和初始状态

- 防止执行恶意代码的安全机制
