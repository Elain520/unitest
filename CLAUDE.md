# CLAUDE.md

此文件为Claude Code (claude.ai/code)在此代码库中工作时提供指导。

## 项目概述

此代码库旨在生成x86汇编测试文件，作为rv环境测试的基准。项目运行在原生x86环境下，用于确定x86代码的正确执行结果，而不是使用模拟器执行x86代码。

## 主要组件

1. **汇编文件** (`*.asm`) - 包含嵌入式JSON配置的x86汇编代码
2. **配置块** - 用`%ifdef CONFIG`和`%endif`指令包围的汇编代码部分，包含JSON格式的测试配置
3. **JSON配置** - 定义寄存器初始状态、内存区域、内存数据和执行模式（32位vs64位）
4. **输出文件** - 生成包含执行后寄存器状态的新汇编文件，作为rv环境测试的基准

## 架构详情

### JSON配置结构
- `RegData`: 输入时应忽略（仅在输出结果中包含最终寄存器状态）
- `Mode`: 执行模式（"32BIT"表示32位，省略表示64位）
- `MemoryRegions`: 执行前要分配的内存区域
- `MemoryData`: 在分配的内存区域中初始化的数据

### 执行流程
1. 加载并解析汇编文件
2. 从CONFIG块中提取JSON配置
3. 解析寄存器初始值和内存设置指令
4. 使用NASM根据Mode配置编译汇编代码
5. 链接目标文件以创建可执行文件
6. 使用指定的内存区域和数据初始化执行环境
7. 在原生x86环境下执行直到HLT指令
8. 获取所有寄存器的最终状态
9. 生成包含RegData部分的输出汇编文件作为执行结果

### 内存管理
- 根据JSON配置分配指定的内存区域
- 根据MemoryData初始化内存内容
- 执行完成后释放所有分配的内存

## 开发指南

### 寄存器处理
- 支持所有16个通用寄存器（RAX-R15）的初始值设置和结果捕获
- 支持XMM寄存器（XMM0-XMM15）的处理
- 支持标志寄存器（EFLAGS/RFLAGS）的处理
- 根据Mode配置确定使用32位或64位寄存器

### 执行环境
- 直接在原生x86硬件或兼容环境中执行
- 不使用任何模拟器（如Box64）
- 确保执行结果的准确性和权威性
- 为rv环境提供可靠的基准测试数据

### 输出格式
- 生成的汇编文件保持与输入文件相同的格式
- CONFIG块中的RegData部分将包含执行后的寄存器状态
- 输出文件可直接用于rv环境的对比测试

### 使用汇编文件
汇编文件遵循以下结构：
1. 在`%ifdef CONFIG`和`%endif`之间的配置部分，包含JSON
2. 以`hlt`指令结尾的常规汇编代码

示例：
```asm
%ifdef CONFIG
{
    "Mode": "32BIT",
    "MemoryRegions": {
        "0x10000000": 4096
    },
    "MemoryData": {
        "0x1000000": ["0x1234567", 1, 3, 5]
    }
}
%endif

mov rdx, 0xc0
shr rdx, 4

hlt
```
