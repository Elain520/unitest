# json基础格式解释：
```json
{
  "RegData": {
    "RAX": "0xc",
    "RBX": "0x0",
    "Flags": "0x0",
    "XMM0": ["0x1","","",""],
    "XMM1": ["",""]
  },
  "RegInit": {
    "RAX": "0x0",
    "RBX": "0x0",
    "Flags": "0x0",
    "XMM0": ["0x0","","",""],
    "XMM1": ["",""]
  },
  "Mode": "32BIT",
  "MemoryRegions": {
    "0x10000000": 4096,
    "0x20000000": "0x2000"
  },
  "MemoryData": {
    "0x100000000" : "9abcdef0 12345678",
    "0x100000020" : "fa aa 55 33",
    "0x100000038" : "0x123456789"
  }
}
```
| 字段 | 描述                                                                                       |
| --- |------------------------------------------------------------------------------------------|
| RegData | 寄存器数据，汇编执行完成后的寄存器数据，在没有Mode字段时，默认为64位寄存器，如果MODE = 32BIT的时候需要考虑32位寄存器，Flags字段为EFLAGS的最终结果 |
| RegInit | 寄存器初始值，汇编执行前的寄存器数据，如果没有该字段，则默认为0|
| Mode | 运行模式，只有一个值，32BIT，如果32BIT存在，则说明当前的运行模式为32位，所有的寄存器都是32位寄存器，并且只应该使用32位寄存器                   |
| MemoryRegions | 内存区域，需要提前申请的内存区域，如果有该字段，在程序运行前需要为该程序申请特定内存，值可以是一个数字或者16进制字符串                             |
| MemoryData | 内存数据，为申请的内存中填入对应的数据，是一段kv对，value是16进制的字符串，可能存在的格式请参考上述内容，key为内存的起始地址                                |
